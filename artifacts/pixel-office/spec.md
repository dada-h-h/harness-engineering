## Overview
Claude Code 에이전트의 활동을 픽셀 아트 오피스 캔버스에 실시간으로 시각화한다.
JSONL 로그를 WebSocket으로 구독하여 에이전트 상태(idle/working/thinking/done)에 따라 캐릭터가 애니메이션되고, 오피스 레이아웃은 에디터로 편집 후 localStorage에 유지된다.

## 시나리오

### 1. WebSocket 연결 성공 — 에이전트 캐릭터 등장
[상황] 사용자가 `/pixel-office` 페이지를 열고 WebSocket 서버(ws://localhost:3001)가 실행 중이면
[동작] 연결이 확립되고 서버로부터 수신된 에이전트 목록이 캔버스에 캐릭터로 표시된다

성공 기준:
- [ ] 연결 성공 후 상태 배지에 '연결됨' 텍스트가 표시된다
- [ ] 서버가 에이전트 1개("agent-1")를 전송하면 캔버스에 캐릭터 1개가 표시된다
- [ ] 서버가 에이전트 3개를 전송하면 캔버스에 캐릭터 3개가 표시된다

### 2. WebSocket 서버 미실행 — 상태 배지 표시
[상황] 사용자가 `/pixel-office` 페이지를 열었으나 WebSocket 서버가 실행 중이 아니면
[동작] 오피스 캔버스는 그대로 보이고 우상단 상태 배지에 '서버 연결 끊김'이 표시된다

성공 기준:
- [ ] 오피스 캔버스(canvas 요소)가 화면에 표시된다
- [ ] 상태 배지에 '서버 연결 끊김' 텍스트가 표시된다
- [ ] 캔버스에 캐릭터가 0개 표시된다

### 3. WebSocket 재연결 — 상태 배지 업데이트
[상황] WebSocket이 끊긴 상태에서 서버가 다시 실행되어 자동 재연결에 성공하면
[동작] 상태 배지가 '연결됨'으로 업데이트되고 에이전트 캐릭터가 다시 표시된다

성공 기준:
- [ ] 재연결 성공 후 상태 배지가 '서버 연결 끊김' → '연결됨'으로 변경된다
- [ ] 서버가 에이전트를 재전송하면 캔버스에 캐릭터가 표시된다

### 4. 에이전트 상태 변화 — 캐릭터 애니메이션 전환
[상황] WebSocket이 연결된 상태에서 서버가 에이전트 상태 업데이트 메시지를 전송하면
[동작] 해당 에이전트의 캐릭터가 대응하는 애니메이션 상태로 전환된다

성공 기준:
- [ ] 상태 `working` 수신 → 해당 캐릭터에 `data-state="working"` 속성이 설정된다
- [ ] 상태 `thinking` 수신 → 해당 캐릭터에 `data-state="thinking"` 속성이 설정된다
- [ ] 상태 `idle` 수신 → 해당 캐릭터에 `data-state="idle"` 속성이 설정된다
- [ ] 상태 `done` 수신 → 해당 캐릭터가 캔버스에서 사라진다(페이드 아웃)

### 5. 레이아웃 에디터 — 가구 배치 저장
[상황] 사용자가 레이아웃 에디터를 열고 가구를 배치한 뒤 저장 버튼을 누르면
[동작] 배치된 레이아웃이 localStorage에 저장된다

성공 기준:
- [ ] '레이아웃 저장' 버튼 클릭 후 '저장됨' 토스트 메시지가 표시된다
- [ ] 가구 배치 후 저장 → localStorage에 `pixel-office-layout` 키로 데이터가 존재한다

### 6. 페이지 재로드 — 레이아웃 복원
[상황] localStorage에 `pixel-office-layout` 데이터가 존재하는 상태에서 페이지를 재로드하면
[동작] 이전에 저장한 가구 배치가 캔버스에 복원된다

성공 기준:
- [ ] 가구 2개를 배치·저장 후 재로드 → 동일한 위치에 가구 2개가 표시된다
- [ ] localStorage 데이터 없이 재로드 → 기본 레이아웃이 표시된다

## 범위

### 포함
- `/pixel-office` 페이지 (Next.js App Router)
- WebSocket 클라이언트 훅 (`useAgentSocket`)
- 연결 상태 배지 (연결됨 / 서버 연결 끊김)
- 픽셀 아트 오피스 캔버스 렌더링
- 에이전트 캐릭터 4가지 상태: idle, working, thinking, done
- 레이아웃 에디터 (가구 drag & drop 배치)
- localStorage 레이아웃 저장/복원
- WebSocket 서버 (`server.ts`): JSONL 감시 + 에이전트 상태 브로드캐스트

### 제외
- JSONL 경로 UI 입력란 (env 변수로 고정하기로 결정)
- 다중 프로젝트 동시 감시 (단일 JSONL_PATH만 지원)
- 인증/보안 (로컬 개발 전용)
- 모바일 반응형 (데스크탑 전용 캔버스)

## 전제 조건
- `JSONL_PATH` 환경 변수가 `.env`에 설정되어 있어야 한다
- WebSocket 서버는 `bun run server`로 별도 실행한다 (포트 3001)
- pixel-agents의 에셋(`public/assets/`)이 복사되어 있어야 한다
- `bun dev`(Next.js)와 `bun run server`가 동시에 실행 중이어야 한다
